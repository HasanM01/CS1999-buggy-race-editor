The main features of the buggy editor are:

The ability to create a buggy in a quick and easy way You can create a user and assign the buggy to yourself Its quick and easy to export the JSON for any buggys that you want to race!

By using my buggy editor you will save yourself the time and stress of picking the parts! Let the majority of the fields be filled by drop downs!

In terms of making further edits, I made the code fairly easy to read and follow along with by having each main function have its own function. Therefore, finding the right function is fairly easy.

Furthermore, the server has a lot of flexibility for additional features to be added in the future, with the CSS being formatted in such a way where further buttons and content can be easily fitted as appropriate.

1-ADD Add more data to the form

I was able to add the flag color variable to the form, which had already been structured into the database.

1-VALID Add basic data validation

I validated the number of wheels field, to only allow integers to be entered.

In the event of a non integer being entered, an error message would display above the input telling the user that only an integer is acceptable.

1-STYLE Style your editor just how you like it

I made changes to the positioning of the buttons

I found initially I was focussing too much on the design and looks of the program (through editing the css) therefore, I chose to move on to the other tasks and implement further functionality to the server

2-EDIT Edit the record by loading its current values into the form

I implemented an edit button in the show buggy area. This allows for the form to show all the currently laid out data and made any modification that you wish.

2-FORM Make the form better

I found it could be tiresome to enter all of the data manually, and adding the relevant validation for each field would also be time-consuming. Therefore, for fields where there are a finite number of results, I added drop downs which removes the chance of adding invalid data. I also modified the show buggy to show the buggies in a table. I added a edit/delete button in the table for each buggy. I experimented in showing the the table transposed.

I kept some fields as data input for convience, as its easier to input the number of wheels manually than it is from a drop down.

2-COST Calculate and save the game cost of the buggy

I hard coded as constants the cost of each part, once the buggy is assembled by the user, the overall price is calculated stored with the buggy, and also displayed with all of the buggy data.

Would have preferred if the costs breakdown could be pulled from server. That way any cost changes would not need the UI code to change.

2-RULES Add validation according to the game rules

Checked all quantities were digits and not strings. Only applied hamster_booster when power_type is hamster. Defined the default values as defined in the specification in the database tables and in the make buggy page.

3-ENV Switch between dev and production environments

Pycharm allows me to adjust the FLASK_ENV when running the program. This is normally set to development and can be changed to production.

3-AUTOFILL Add auto-fill to the edit forms

I chose to further enhance and add further drop down menu's as I found them easier for data input rather than input boxes.

3-MULTI Allow different buggies to be created

I changed the way in which the system worked, each buggy is assigned an id, so when a new buggy is created then a new database entry is made which allows for them all shown.

3-DEL Allow buggies to be deleted

Along with the edit button in the show buggy screen I added a delete button.

The process of getting the edit button and delete button to format correctly took some time, as the table would sometimes misallign.

4-USERS Add users (and sessions) so you know who is editing a buggy

Created a users table in the data and add a user_id column to the buggy database. Additionally changed the buggy page so user can be assigned to a buggy. Drop down list shows user name, which is better than showing user id.

4-REGISTER Make a new user

Implemented a add user page and page to show users. Each new user added is assigned a new id and all users as a table in show users page

4-OWNER A buggy belongs to a user

When making a buggy or editing a buggy, a user can be selected. Implemented a drop down of user name available
